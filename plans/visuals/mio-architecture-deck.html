MIO · Engineering Brief · 2026-05-06

Channels are messy. Agents shouldn't care.

MIO is the messaging I/O platform for MIU — one canonical envelope across Cliq, Slack, Discord, Telegram. POC with ABSpectrum on Zoho Cliq, deploying to GKE this week.

The mismatch

A 3-second webhook ack and a 30-second LLM call cannot share a thread.

Slack and Discord drop the message if you don't ack in 3s. Coupling the webhook handler to the LLM call means dropped messages on the first slow agent run.

Concept · canonical envelope

One envelope. Two streams. Pure intelligence on the other side.

// mio-proto · Message (inbound) message Message { string id = 1; // ulid string source_msg_id = 2; // channel id Channel channel = 3; // SLACK | CLIQ | … string workspace_id = 4; string thread_id = 5; // opaque User sender = 6; string text = 7; repeated Attachment attachments = 8; int64 ts_ms = 9; } // mio-proto · SendCommand (outbound) message SendCommand { string idempotency_key = 1; Channel channel = 2; string reply_to_id = 3; string text = 4; }

Inbound

Channel → Message

HMAC verified, payload normalized into a Message proto. Channel-specific IDs preserved as opaque strings.

Outbound

SendCommand → Channel

AI returns a SendCommand. Gateway translates back to channel REST. No SDK leaks past the bus.

Bus

Durable spine

Two streams: MESSAGES_INBOUND 7d replay, MESSAGES_OUTBOUND 24h workqueue.

Architecture · NATS JetStream layout

NATS JetStream layout — two streams, multiple durable consumers.

INBOUND limits retention 7d · replayable  ·  OUTBOUND workqueue 24h ceiling

3s  vs  30s

Webhook ack deadline vs typical LLM round-trip. The bus is the only place that contradiction can live without losing messages.

ADR · locked in

Three decisions hold the rest of the system up.

ADR-001

Bus: NATS JetStream

nats · 3-replica · GKE

Why: ~50MB idle, K8s-native, cloud-agnostic. Kafka is a sledgehammer; Redis Streams loses durability we need for replay. JetStream gives both, on the cluster we already run.

ADR-002

Languages: Go + Python

gateway · go  ·  ai · python

Why: Channel SDKs (slack-go, discordgo) mature in Go; LangGraph is Python-first. Don't fight the ecosystem. Wire them with proto, not REST.

ADR-003

Schema: Protobuf via buf

mio-proto · breaking = new pkg

Why: JSON between Go and Python drifts within a month. buf gives codegen, lint, breaking-change detection. Versioned envelope or none at all.

First channel · validated POC

Zoho Cliq for ABSpectrum: the proving ground for the abstraction.

Design rules · non-negotiable

The rules that keep the gateway dumb and the agents honest.

Gateway is dumb

Verify signature, normalize, publish, ack. Zero LLM logic, zero business rules, zero retries that cost money. Stateless = horizontal scale is free.

Idempotency at the edge

Unique on (channel, source_message_id). Cliq is at-least-once; we silently dedupe. Outbound carries a client-generated ULID for the same reason.

Per-workspace rate limits

Token bucket per tenant, never global. One chatty workspace cannot starve another. Per-thread ordering via MaxAckPending=1 on the AI consumer to start.

Plan · POC roadmap

Five passes from "Cliq POC" to "ABS GKE serving downstream".
